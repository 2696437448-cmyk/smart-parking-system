## 毕业论文（设计）题目

中文题目：智慧社区多源数据融合的车位动态调度与共享系统设计

英文题目：Design of a Dynamic Parking Dispatch and Sharing System for Smart Communities Based on Multi-Source Data Fusion

## 一、毕业论文（设计）的主要内容

本课题面向社区停车管理场景，围绕车位状态更新不及时、临时共享缺少统一调度、住户停车流程不够连贯、物业端缺少经营分析支撑等问题，设计并实现一套智慧社区车位动态调度与共享系统。系统整体采用微服务加前后端分离的实现方案，后端由网关服务、停车主业务服务、模型服务和实时服务组成，前端提供业主端和物业端两个主要使用界面。

课题的主要内容包括以下几个方面。第一，围绕车位状态、车辆进出记录和住户出行规律等多源数据进行整合处理，为后续预测、调度和经营统计提供基础。第二，采用轻量级 LSTM-Lite 预测方案，对停车供需变化进行估计，为推荐和管理分析提供参考。第三，采用 Hungarian 匹配方法实现动态车位分配，使系统能够在约束条件下完成请求与车位资源的合理匹配。第四，完成业主端推荐、预约、订单、计费和导航等主要业务流程，形成较完整的使用闭环。第五，完成物业端经营摘要、收益趋势、占用率趋势、预测对照和实时状态说明等功能。第六，在系统实现过程中同步考虑幂等控制、锁机制、消息重试、服务降级和实时回退等工程问题，使系统具备较好的稳定性和验收基础。

## 二、毕业论文（设计）的基本要求/技术指标

1. 论文题目、系统功能和正文内容应保持一致，研究主线围绕社区智慧停车、车位调度与共享展开。
2. 系统应体现前后端分离和服务分层思路，模块职责划分清晰，能够说明各服务之间的关系。
3. 数据处理部分应体现多源数据融合特点，能够说明车位状态、车辆进出和住户行为数据如何支撑后续业务。
4. 预测模块与调度模块应形成衔接关系，能够说明 LSTM-Lite 预测结果如何辅助车位推荐与经营分析，Hungarian 方法如何支撑动态分配。
5. 业主端应具备推荐、预约、订单、计费和导航等主要流程，物业端应具备经营监控、趋势展示和状态说明等功能。
6. 系统应考虑基本的可靠性设计，包括请求幂等、并发控制、异常降级、实时通道回退和链路可追踪性。
7. 论文正文应完整说明研究背景、技术路线、系统设计、实现过程、测试与结果分析、结论与展望等内容。
8. 文本表达应符合本科毕业设计要求，内容具体、结构清晰、结论谨慎，不夸大系统效果。
