# 项目内术语表

## Step40

当前默认完成态。学习、演示和默认验收都以它为准。

## Composable

前端里把页面逻辑抽离出来的函数层。页面负责展示，composable 负责数据和状态编排。

## Gateway

系统统一入口。它根据路径把请求转发给不同后端服务。

## Controller

后端 HTTP 入口层，负责接请求和回响应，但不一定承载全部业务逻辑。

## Query / Assembler / ViewService

Dashboard 聚合时常见的三层分工：查询数据、组装页面结构、对外提供完整视图。

## View-model

专门为页面准备的返回结构，不等于数据库原表。

## ETL

把原始数据清洗、整理、转换成后续建模和调度能直接使用的结构。

## Registry

模型注册表，用来记录有哪些模型、当前激活哪个版本，而不是把版本写死在代码里。

## Realtime 降级

WebSocket 不可用时，页面退回到 polling 继续工作，而不是整页失效。

## Idempotency

幂等语义。同一请求被重复提交时，系统尽量保证结果一致、不重复创建资源。
