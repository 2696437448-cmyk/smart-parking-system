# 术语表

这份术语表只解释当前学习包里最常见、最容易把新手卡住的词。默认都用 `smart-parking-thesis` 这个项目语境来解释。

## 微服务

把系统拆成多个职责清晰的服务。  
在这个项目里，前端、网关、主业务、模型服务、实时服务是分开的。

## 网关 Gateway

系统的统一入口。  
它负责接收请求、转发到后端服务，并处理 trace、fallback、CORS 等问题。

## Trace ID

一次请求的编号。  
它会从前端一路带到网关和后端，方便排查问题。

## Fallback

兜底方案。  
比如模型服务挂了，网关可以返回一个降级结果，而不是直接把整个页面打崩。

## CORS

浏览器跨域访问控制。  
简单理解就是：浏览器默认不让网页随便请求别的地址，服务端需要显式允许。

## Controller

后端 HTTP 入口层。  
主要负责接收请求和返回响应。

## Service

后端业务逻辑层。  
主要负责“规则到底怎么跑”。

## Repository

后端数据访问层。  
主要负责数据库读写。

## 幂等 Idempotency

同样的请求多次执行，结果应该保持一致，不应该反复创建新资源。  
在这个项目里，预约和订单完成都用了幂等设计。

## 分布式锁

一种防止并发冲突的机制。  
在这个项目里用于避免同一车位在同一时间窗口被重复预订。

## 唯一约束

数据库层面的保护。  
即使应用层漏掉了，数据库也能阻止某些重复写入。

## View-Model

专门为页面准备的数据结构。  
它不一定等于数据库结构，而是“页面最方便使用的结构”。

## Assembler

组装器。  
它负责把查询结果整理成页面更容易消费的 view-model。

## Dashboard 聚合

把多个经营数据来源组装成一个页面需要的统一接口返回。  
这样前端不用自己拼很多接口。

## WebSocket

一种双向通信方式。  
在这个项目里主要用来做物业页面的实时状态推送。

## Polling

轮询。  
前端每隔一段时间主动发一次请求来拿最新数据。  
这里是 WebSocket 降级时的备用方案。

## Degraded / 降级

部分能力不可用时，切换到兼容方案继续工作。  
例如 WebSocket 不可用，就切到 polling。

## ETL

把原始数据提取、清洗、转换成更适合建模或分析的数据表。

## LSTM-lite

项目里使用的一种轻量版时序预测模型实现。  
重点是工程可用、依赖轻、能解释，不是追求最复杂的学术模型。

## Hungarian 算法

常用于分配问题的优化算法。  
在这个项目里，用来做请求和车位之间的较优匹配。

## Model Registry

模型注册表。  
记录有哪些模型、当前激活哪个版本、历史切换记录是什么。

## OpenAPI

接口合同说明文档。  
告诉前后端请求参数和返回结构应该长什么样。

## Acceptance

验收。  
表示项目不是只靠肉眼看页面，而是有一组脚本和规则来验证当前完成态。
